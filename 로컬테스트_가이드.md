# 🧪 로컬 테스트 가이드 — 루밍 PC에서 PDF 생성 검증

> 목적: Streamlit 배포 전에 PC에서 PDF가 진짜 잘 만들어지는지 확인.
> 환경: Windows + Python 3.x

---

## STEP 0. 필요한 것 체크리스트

테스트 시작 전 PC에 이 4가지가 있는지 확인:

- [ ] **Python 3.9 이상** 설치 (확인: 명령프롬프트에서 `python --version`)
- [ ] **나눔 폰트 4개** 위치 확인
  ```
  C:\Users\이설빈\AppData\Local\Microsoft\Windows\Fonts\
    ├ NanumGothic.ttf
    ├ NanumGothicBold.ttf
    ├ NanumMyeongjo.ttf
    └ NanumMyeongjoBold.ttf
  ```
- [ ] **카드 이미지 79장** 위치 확인
  ```
  C:\Users\이설빈\Desktop\tarot_pdf\
    ├ major_00.png.jpeg ~ major_21.png.jpeg
    ├ wands_A.png.jpeg, wands_2 ~ wands_K.png.jpeg
    ├ cups_A.png.jpeg ~ cups_K.png.jpeg
    ├ swords_A.png.jpeg ~ swords_K.png.jpeg
    ├ pentacles_A.png.jpeg ~ pentacles_K.png.jpeg
    └ card_back.png.jpeg
  ```
- [ ] **Claude가 만든 코드 폴더** 다운로드 받아둠 (`/mnt/user-data/outputs/`)

## STEP 1. 작업 폴더 만들기

루밍이 편한 위치에 새 폴더를 만들어. 예: `C:\Users\이설빈\Desktop\kmong-tarot\`

그 안에 다음 구조 만들기:

```
kmong-tarot/
├── manse_calc.py            ← Claude가 준 파일
├── lucky_items.py           ← Claude가 준 파일
├── astro_items.py           ← Claude가 준 파일
├── extract_cards.py         ← Claude가 준 파일
├── make_backup_docx.py      ← Claude가 준 파일
├── make_daily.py            ← Claude가 준 파일
├── make_weekly.py           ← Claude가 준 파일
├── make_monthly.py          ← Claude가 준 파일
├── cards_db.json            ← Claude가 준 파일 (1.6MB)
├── fonts/                   ← 새로 만들고 폰트 4개 복사
│   ├── NanumGothic.ttf
│   ├── NanumGothicBold.ttf
│   ├── NanumMyeongjo.ttf
│   └── NanumMyeongjoBold.ttf
├── images/                  ← 새로 만들고 카드 이미지 79장 복사
│   ├── major_00.png.jpeg
│   ├── ... (79장)
│   └── card_back.png.jpeg
└── output/                  ← 자동 생성됨. PDF 결과물 저장 위치
```

### 실행할 명령 (명령프롬프트에서)

```cmd
cd C:\Users\이설빈\Desktop\kmong-tarot
mkdir fonts
mkdir images
```

그리고:
1. Claude가 준 8개 .py 파일 + cards_db.json을 `kmong-tarot/` 폴더에 복사
2. `C:\Users\이설빈\AppData\Local\Microsoft\Windows\Fonts\`의 나눔 폰트 4개를 `fonts/`에 복사
3. `C:\Users\이설빈\Desktop\tarot_pdf\`의 카드 이미지 79장을 `images/`에 복사

## STEP 2. 파이썬 라이브러리 설치

명령프롬프트에서 (`kmong-tarot/` 폴더 안에서):

```cmd
pip install reportlab korean-lunar-calendar python-docx
```

성공하면 다음 줄이 나옴:
```
Successfully installed reportlab-X.X korean-lunar-calendar-0.3.1 python-docx-X.X
```

### 만약 오류가 나면

- **"pip is not recognized"** → Python을 설치할 때 PATH 옵션을 안 켰을 가능성. Python 재설치 시 "Add to PATH" 체크.
- **권한 오류** → 명령프롬프트를 "관리자 권한으로 실행" 하거나, `pip install --user` 사용.


### 그 다음 — PDF 직접 열어서 시각 검증

`output/` 폴더에 생성된 PDF를 더블클릭으로 열고 다음 항목 확인:

#### ✅ 페이지별 체크리스트

**p1 (카드 페이지)**
- [ ] 한글이 깨지지 않고 정상 표시
- [ ] 제목 "홍길동님의 오늘의 운세"가 보임
- [ ] 카드 이미지가 가운데에 표시됨 (이미지 없으면 박스만 보임)
- [ ] 카드명 "The Sun (태양) — 정방향" 같이 영문+한글 표시
- [ ] MBTI × 카드 본문이 한 페이지에 잘 들어감
- [ ] **"오늘의 핵심"** 골드색 강조 표시 정상

**p2 (별자리/띠 페이지)**
- [ ] 별자리 에너지 텍스트 자연스러운지 (받침/조사 검증)
- [ ] 띠 포인트 텍스트 자연스러운지
- [ ] 에너지 지도 표 4행 (일/관계/금전/건강) 정상
- [ ] ★/☆ 별점 정상 표시
- [ ] 주의 포인트 표시됨

**p3 (행운 아이템 페이지)**
- [ ] 행운 아이템 10개 표가 2열로 깔끔하게 표시
- [ ] 행운의 숫자 2개 정상
- [ ] 확언 3문장 표시됨

**p4 (종합 리딩)**
- [ ] 4가지 교차 요약 표 정상
- [ ] 핵심 키워드 강조
- [ ] 주의 신호 표시
- [ ] 오늘 할 것 한 가지

**푸터**
- [ ] 모든 페이지 하단에 `루밍의 MBTI 타로 운세 | 일간 운세 | 홍길동님 전용 | n/4` 표시
- [ ] 페이지 번호가 정확히 1/4 ~ 4/4

## STEP 6. 실제 고객 정보로 테스트

기본 테스트가 OK면, 실제 고객 정보로 한 번 만들어보기.

**make_daily.py 마지막 부분 수정** (또는 새 파일 만들기):

```python
# 새 파일: test_real_customer.py
from make_daily import make_daily_pdf
from datetime import date

customer = {
    'name': '실제고객이름',
    'mbti': 'ENFP',                # 실제 MBTI
    'birthdate': '1995-08-20',     # 실제 생년월일
    'jisi': '오시',                 # 12지시 (모르면 None)
}

pdf = make_daily_pdf(customer, date_obj=date(2026, 5, 13))
print(f"생성됨: {pdf}")
```

```cmd
python test_real_customer.py
```

## 🔴 자주 발생할 오류 + 해결법

### 오류 1: `ModuleNotFoundError: No module named 'reportlab'`
→ pip install이 안 됐음. STEP 2 다시.

### 오류 2: 한글이 ■ 로 깨짐
→ `fonts/` 폴더에 폰트 4개 확실히 있는지 확인. 파일명 정확히 `NanumGothic.ttf` (대소문자).

### 오류 3: 카드 이미지가 안 보이고 빈 박스만 표시
→ `images/` 폴더에 79장 다 있는지 확인. 파일명 규칙: `major_00.png.jpeg` 식.

### 오류 4: `FileNotFoundError: cards_db.json`
→ `cards_db.json`이 `kmong-tarot/` 폴더에 있는지 확인.

### 오류 5: `KeyError: 'major_XX_upright'`
→ cards_db.json 손상. Claude가 준 원본으로 다시 받아 교체.

### 오류 6: PDF가 열리는데 페이지 중간에 잘림
→ KeepTogether 미적용 부분 있을 수 있음. Claude한테 어떤 페이지/어떤 박스인지 알려주면 수정.

### 오류 7: 푸터가 안 보임
→ NumberedCanvas 미적용 의심. Claude한테 알려줘.

### 오류 8: 역방향 카드 이미지가 정방향으로 나옴
→ **이건 알려진 미구현 항목**. 다음 단계에서 PIL로 회전 처리 추가 예정.

## 📋 테스트 후 보고할 사항

테스트 끝나면 Claude한테 알려줄 정보:

1. **3종 PDF 다 생성됐는가** (일간/주간/월간)

3. **에러 메시지 있으면 그대로 복사**
4. **개선하고 싶은 부분** (디자인, 톤, 추가하고 싶은 내용 등)

→ 이걸 받으면 Claude가 수정하고 다음 단계로 진행.
